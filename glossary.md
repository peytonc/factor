# Mathematical Glossary

Defines the notation used across the theorem and proof files.
Symbols are **not** re-defined elsewhere; every file references this document.

> Convention: `v[i]` denotes the i-th component (0-indexed) of a state vector `v`.
> `^T` denotes transpose. All matrices act on column vectors from the left.

## Euclid Parameters

### `m` ‚Äî first Euclid parameter
- **Definition:** `m = (q + p) / 2 = (sqrt(c+b) + sqrt(c-b)) / 2`.
- **Constraints:** `m > n > 0`, `gcd(m, n) = 1`.

### `n` ‚Äî second Euclid parameter
- **Definition:** `n = (q - p) / 2 = (sqrt(c+b) - sqrt(c-b)) / 2`.
- **Constraints:** `m > n > 0`, `gcd(m, n) = 1`.

## Factors and Triple Legs

### `p` ‚Äî smaller factor of the odd leg
- **Definition:** `p = m - n = sqrt(c - b)`.
- **Constraints:** `1 < p < q` (on non-trivial nodes; `p = 1` on the trivial A-branch).

### `q` ‚Äî larger factor of the odd leg
- **Definition:** `q = m + n = sqrt(c + b)`.
- **Constraints:** `1 < p < q`.

### `a` ‚Äî odd leg
- **Definition:** `a = m^2 - n^2 = p * q`.
- **Notes:** the quantity being matched against `a_target`.

### `b` ‚Äî even leg
- **Definition:** `b = 2mn = (q^2 - p^2) / 2`.

### `c` ‚Äî hypotenuse
- **Definition:** `c = m^2 + n^2 = (p^2 + q^2) / 2`.

## Targets and Unknowns

### `a_target`
- **Definition:** a large, known, odd **semiprime** representing the odd leg `a`.
- **Notes:** its tree depth is extremely large, so only shallow search is feasible.

### `m_target`, `n_target`, `p_target`, `q_target`
- **Definition:** the unknown integers locating `a_target` in the tree, with
  `a_target = p_target * q_target` and `1 < p_target < q_target`.

## State Vector

### `v` ‚Äî algebraic state vector of a PPT node
- **Definition:** `v = [p^2, q^2, a]^T` in Z^3.
- **Components:** `v[0] = p^2`, `v[1] = q^2`, `v[2] = a`.
- **Invariant:** `v[0] * v[1] = v[2]^2` (i.e. `p^2 * q^2 = (pq)^2`).

### `v_root` ‚Äî projected root of the PPT tree
- **Definition:** `v_root = [1, 9, 3]^T` (the triple with `p = 1, q = 3, a = 3`).

## Berggren (Geometric) Generators

Operate on the geometric triple `(a, b, c)`.

### `Berggren_A`
- **Definition:** `[[1, -2, 2], [2, -1, 2], [2, -2, 3]]`.

### `Berggren_B`
- **Definition:** `[[1, 2, 2], [2, 1, 2], [2, 2, 3]]`.

### `Berggren_C`
- **Definition:** `[[-1, 2, 2], [-2, 1, 2], [-2, 2, 3]]`.

## Factor-State Generators

Integer matrices acting on the algebraic state vector `v = [p^2, q^2, a]^T`.

### `A` ‚Äî factor-state matrix for Berggren_A
- **Definition:** `[[1, 0, 0], [4, 1, 4], [2, 0, 1]]`.
- **Properties:** preserves `p`, increases `a`, `det(A) = 1`.

### `B` ‚Äî factor-state matrix for Berggren_B
- **Definition:** `[[0, 1, 0], [1, 4, 4], [0, 2, 1]]`.
- **Properties:** promotes old `q` to new `p`, increases `a`, `det(B) = -1`.

### `C` ‚Äî factor-state matrix for Berggren_C
- **Definition:** `[[0, 1, 0], [1, 4, -4], [0, 2, -1]]`.
- **Properties:** promotes old `q` to new `p`, increases `a`, `det(C) = 1`.

### `G` ‚Äî a selected generator
- **Definition:** `G in {A, B, C}`, a state-transition matrix chosen at a step.

## Paths and Sequences

### `P` ‚Äî generator sequence to depth d
- **Definition:** `P = G_1 * G_2 * ... * G_d` with each `G_i in {A, B, C}`, applied
  to `v_root` to reach depth `d`.
- **Note:** matrix multiplication order is the **reverse** of path/traversal order.

### `d` ‚Äî tree depth
- **Definition:** the number of generator applications from the root.

## Modular Objects

### `M` ‚Äî modulus
- **Definition:** an odd integer modulus, typically `3 <= M < sqrt(a_target)`.

### `Orbit(M)` / `O_M`
- **Definition:** the set of reachable states `{ gamma . v_root (mod M) : gamma in Gamma }`.

### `Gamma`
- **Definition:** the group `<A, B, C>` generated by the transition matrices in `GL(3, Z/MZ)`.

### `Phi`
- **Definition:** the canonical CRT projection `O_M -> prod O_(component_i)`.

## Differential Objects

### `delta_a`, `delta_b` ‚Äî sibling-divergence vectors
- **Definition:** `delta_a = [0, 8, 2]^T` (B vs C); `delta_b = [2, 6, 4]^T` (B vs A).

### `e_a`, `e_b` ‚Äî half-divergence vectors
- **Definition:** `e_a = delta_a / 2 = [0, 4, 1]^T`; `e_b = delta_b / 2 = [1, 3, 2]^T`.

### `a_parent`, `b_parent` ‚Äî parent leg scalars
- **Definition:** `a_parent = v_parent[2]`; `b_parent = (v_parent[1] - v_parent[0]) / 2`.

### `alpha` (`alpha_m`) ‚Äî localized parent scalar
- **Definition:** the relevant parent leg at a swap: `a_parent` for B<->C swaps,
  `b_parent` for A<->B swaps.

### `M_m` ‚Äî modular base scalar
- **Definition:** `M_m = 2 * alpha_m`.

### `sigma` (`sigma_m`) ‚Äî directional swap sign
- **Definition:** `+1` for transitions `(C -> B)` or `(A -> B)`; `-1` for `(B -> C)` or `(B -> A)`.

### `R` (`R_m`) ‚Äî suffix matrix product
- **Definition:** the product of all generators applied **after** the swap depth, down
  to the leaf (`R = G_leaf * ... * G_(d+2)`), preserving right-to-left multiplication order.

### `lambda` ‚Äî factor ratio
- **Definition:** `lambda = q / p` for a node with known factors (used in T16 for
  deterministic parent identification).

### `Lambda` ‚Äî divergence lattice
- **Definition:** the integer lattice `Z-span{ M_m * (R_m * e_m) }` containing the exact
  multi-swap leaf difference (T17).
